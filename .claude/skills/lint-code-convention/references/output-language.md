# Output Language

## Description

All content generated by the Flow plugin must follow the language specified in the project's CLAUDE.md harness section (`### Language` field). If no language is specified, default to English. This ensures consistent cross-referencing across all documents while supporting multilingual projects.

## Rules

### OL-1: Generated artifacts must follow the configured language

All files produced during the workflow must use the language specified in the CLAUDE.md harness section (`### Language` field). If no language setting is found, default to English.

Affected files:
- Harness documents (`harness/*.md`)
- Meeting logs (`harness/topics/*/meetings/*.md`)
- CPS documents (`harness/topics/*/cps.md`)
- PRD documents (`harness/topics/*/prd.md`)
- Design documents (`harness/topics/*/spec.md`, `blueprint.md`, etc.)

Files that are always in English regardless of language setting:
- Lint skill definitions (`.claude/skills/lint-*/SKILL.md`)
- Lint skill references (`.claude/skills/lint-*/references/*.md`)
- Agent prompts (`agents/*.md`)
- Skill prompts (`skills/*/SKILL.md`)
- Code comments in source files

### OL-2: Agent and skill prompts must be in English

All files in `agents/` and `skills/` must be written in English. These are consumed by LLM agents and must be unambiguous. This rule applies regardless of the configured language setting.

**Known violations (existing, tracked in tech-debt.md):**
- `skills/using-worktree/SKILL.md` contains Korean text in the "When This Applies" section
- `agents/meeting-facilitator.md` uses Korean section headers in meeting log templates (e.g., "Discussion", "Decisions", "Unresolved Items", "Resolved Items" have Korean labels)
- `README.md` is primarily in Korean (acceptable as user-facing documentation, but noted for awareness)

### OL-3: Code comments may use English only

Inline comments in JavaScript and shell scripts must be in English.

**Detection approach:**
Scan `.md`, `.cjs`, `.js`, and `.sh` files for runs of CJK characters (Unicode ranges U+3000-U+9FFF, U+AC00-U+D7AF) that indicate non-English content. Exclude README.md from automated checks (user-facing documentation may be bilingual by policy).

### OL-4: Error messages in English

All error messages output by scripts and agents must be in English.

**Correct:** `{"error": "Server failed to start within 5 seconds"}`
**Incorrect:** `{"error": "서버가 5초 안에 시작되지 않았습니다"}`
