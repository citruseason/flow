# Impeccable: Design Skills for AI Development Tools

Enhanced frontend-design skill with anti-patterns. Provides design vocabulary and commands for building polished, professional interfaces.

## The Problem

Great design prompts require design vocabulary. Most developers don't have it. This skill bridges that gap by providing structured design knowledge across typography, color, layout, and motion.

## Installation

This skill references Impeccable's design system. For the full design command set, install the Impeccable package:

```bash
npx skills add pbakaus/impeccable
```

Or install as a Claude Code plugin via `/plugin marketplace`.

## Core Design Commands

| Command | Purpose |
|---------|---------|
| `/polish` | Apply design polish to existing UI |
| `/audit` | Audit design quality and find issues |
| `/typeset` | Improve typography and text hierarchy |
| `/overdrive` | Maximum design enhancement mode |

## Design Principles

### Typography
- Establish clear type hierarchy (display, heading, body, caption)
- Use consistent scale ratios
- Ensure readable line lengths (45-75 characters)
- Apply proper vertical rhythm

### Color
- Build from a systematic palette (primary, neutral, semantic)
- Ensure WCAG contrast compliance
- Use color purposefully, not decoratively

### Layout
- Use consistent spacing scale (4px/8px base)
- Apply whitespace intentionally for visual breathing room
- Create clear visual hierarchy through size, weight, and spacing

### Motion
- Use animation to communicate, not decorate
- Keep transitions under 300ms for responsiveness
- Respect prefers-reduced-motion

## Key Design Anti-Patterns

- Generic placeholder content with no visual hierarchy
- Equal visual weight on all elements (no focal point)
- Inconsistent spacing and alignment
- Missing hover/focus states
- Typography that doesn't guide the eye

## What Good Design Looks Like

Every element serves a purpose. Hierarchy guides the eye. Whitespace breathes. Interactions feel responsive and intentional.
