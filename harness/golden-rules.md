# Golden Rules

These are core invariants for this project. Violating any of these should block a merge.

## Architecture

- **Agent-skill separation**: Agents execute work (analysis, writing, reviewing). Skills orchestrate (dispatch agents, manage confirmation gates, update kanban, handle git commits). An agent must never perform orchestration tasks; a skill must never contain execution logic.
- **Dependency direction**: Skills may reference other skills by slash command. Agents must not reference other agents. Scripts must not reference agents or skills. The dependency flow is: skills -> agents -> references, never reverse.
- **Self-contained skills**: Every skill directory must contain a SKILL.md with valid YAML frontmatter (name, description). Optional references/ directory for supporting documents. No skill may depend on files outside its own directory or the harness/.

## Model Assignment

- **Writer agents use Opus**: harness-initializer, meeting-facilitator, design-doc-writer -- all agents that create or generate content must use `model: opus`.
- **Reviewer agents use Sonnet**: meeting-reviewer, design-doc-reviewer, doc-gardener, lint-reviewer -- all agents that validate or review must use `model: sonnet`.

## Version Sync

- **plugin.json and marketplace.json versions must match**: `.claude-plugin/plugin.json` version field must equal `.claude-plugin/marketplace.json` plugins[0].version at all times. Both files must be updated together.
- **Patch-only versioning**: Until 1.0.0, only increment the patch version (0.0.1 -> 0.0.2 -> ...).

## Output Language

- **All generated output in English**: Every file generated by agents and skills must be written in English regardless of the user's language. This applies to harness/ documents, lint-* skills, meeting logs, CPS, PRD, design documents, and all other generated artifacts.

## Pipeline Integrity

- **Sequential pipeline**: The workflow proceeds in order: harness-init -> meeting -> design-doc -> implement -> lint. No step may be skipped. Each step's outputs are inputs to the next.
- **Autonomous execution boundary**: From /implement start through /lint completion, execution proceeds without user intervention. Only escalate on severe mismatches or unresolvable blockers.
- **User gates exist at meeting and design-doc**: Users must approve CPS, PRD, and each design document before proceeding. These gates must not be bypassed.

## Document History

- **FIFO rotation with max 2 versions**: When archiving documents to history/, use consistent FIFO rotation: v1 = most recent prior version, v2 = older version. Maximum 2 archived versions per document.
