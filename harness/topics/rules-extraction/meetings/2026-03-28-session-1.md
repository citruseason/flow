## Decisions

1. PRODUCT.md는 순수하게 프로젝트 정의(개요, 스택, 에이전트/스킬 테이블, 문서 흐름, 링크)만 포함
2. 아키텍처/파이프라인/관찰 가능성은 harness/ 루트에 CORE 문서로 분리 (ARCHITECTURE.md, PIPELINE.md, OBSERVABILITY.md)
3. 컨벤션 규칙은 harness/rules/ 하위에 도메인+스택별 파일로 분리 (`{도메인}.{스택}.md`)
4. PRODUCT.md에서 분리된 문서들을 링크로 참조
5. lint-* 스킬의 upstream 참조를 새 파일 경로로 업데이트
6. CLAUDE.md 하네스 섹션도 새 CORE 문서 반영
7. harness-initializer 에이전트가 새 구조를 생성하도록 업데이트
