# Meeting Log: Hooks Integration

- **Date:** 2026-03-28
- **Topic:** hooks-integration
- **Session:** 1

## Discussion

### 조사: Claude Code Hook 시스템
- 26가지 hook 이벤트 지원 (SessionStart, PreToolUse, PostToolUse, Stop, SubagentStop, FileChanged, UserPromptSubmit 등)
- 현재 Flow 사용: SessionStart 환영 메시지만
- 7개 활용 포인트 식별 (높은 가치 4개 + 중간 가치 3개)

### Q1: 구현 범위
- 선택지: (a) 높은 가치 4개 (b) 1,2번만 (c) 전부 (d) 직접 선택
- **결정:** (a) 높은 가치 4개 모두 구현
  1. PostToolUse — 머지 감지 → core-update
  2. SessionStart — 세션 복원 시 컨텍스트 재주입
  3. Stop — 자율 실행 완료 후 kanban 요약
  4. SubagentStop — 에이전트 완료 시 kanban 자동 업데이트

### Q2: 머지 감지 구현 범위
- 선택지: (a) 알림만 (b) 자동 실행 (c) 확인 후 실행
- **결정:** (b) 자동 실행 — 머지 감지 시 /core-update 자동 트리거

### Q3: 세션 복원 컨텍스트
- 선택지: (a) 토픽 상태만 (b) + harness 요약 (c) + 마지막 작업 컨텍스트
- **결정:** (c) 토픽 상태 + harness 요약 + 마지막 작업 컨텍스트 (수정 파일, 마지막 커밋)

### Q4: SubagentStop kanban 자동화
- 선택지: (a) hook으로 완전 대체 (b) 보조 역할 (c) Stop에 통합
- **결정:** (a) hook으로 완전 대체 — SubagentStop hook이 kanban.json 직접 업데이트, 스킬에서 kanban 로직 제거

## Decisions

1. 4개 hook 구현: PostToolUse(머지), SessionStart(복원), Stop(요약), SubagentStop(kanban)
2. 머지 감지는 자동 실행 (사용자 확인 없이 /core-update 트리거)
3. 세션 복원은 토픽 + harness + 마지막 작업 컨텍스트 모두 재주입
4. SubagentStop hook이 kanban.json을 직접 업데이트하여 스킬의 kanban 로직 대체
